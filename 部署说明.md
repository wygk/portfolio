# 部署说明 · 将作品集发布到 GitHub Pages

本目录即为可直接发布的作品集仓库内容。发布后即可获得公开链接，填入简历。

## 目录结构

```
作品集/
├── README.md        # 作品集主页（GitHub Pages 首页 + 仓库首页都用它）
├── _config.yml      # GitHub Pages 主题配置（Cayman）
├── 殷刚_简历.md      # Markdown 版简历（已含作品集链接，可再导出 PDF）
└── 部署说明.md       # 本文件
```

## 一、准备工作

1. 注册并登录 [GitHub](https://github.com)，你的用户名为 `wygk`。
2. 本机安装 [Git](https://git-scm.com/download/win)。

## 二、创建仓库

1. GitHub 右上角 **+ → New repository**。
2. **Repository name** 填：`portfolio`
3. 选择 **Public**（GitHub Pages 免费版需公开仓库）。
4. 不要勾选任何初始化选项，点击 **Create repository**。

## 三、推送本地文件（在「作品集」目录下执行）

> PowerShell 中命令分隔请用 `;`，不要用 `&&`。

```powershell
cd "E:\work\文档\个人\作品集"
git init
git add .
git commit -m "add portfolio"
git branch -M main
git remote add origin https://github.com/wygk/portfolio.git
git push -u origin main
```

## 四、开启 GitHub Pages

1. 进入仓库 **Settings → Pages**。
2. **Source** 选择 `Deploy from a branch`。
3. **Branch** 选择 `main`，目录选 `/ (root)`，点击 **Save**。
4. 等待约 1 分钟，页面顶部会显示已发布链接：

```
https://wygk.github.io/portfolio/
```

## 五、把链接放入简历

- `殷刚_简历.md` 顶部已预置作品集链接（https://wygk.github.io/portfolio/），直接用编辑器（如 Typora / VS Code）导出为 PDF，即可替换原 `殷刚_简历.pdf`。
- 若想保留原 PDF 排版：用 PDF 编辑器（如 WPS、Adobe、[pdfescape.com](https://www.pdfescape.com)）在联系方式区域添加一行：
  `作品集：https://wygk.github.io/portfolio/`

## 六、后续更新

修改任意文件后，在「作品集」目录执行：

```powershell
git add .
git commit -m "update"
git push
```

约 1 分钟后线上页面自动更新。

---

### 备选：不想用 GitHub Pages？

- **只要一个可分享的链接**：直接把仓库地址 `https://github.com/wygk/portfolio` 放进简历，GitHub 会自动渲染 `README.md`。
- **想要更漂亮的独立站点**：可将本目录部署到 Vercel（本环境支持 vercel-deploy 技能，如需可再协助）。
